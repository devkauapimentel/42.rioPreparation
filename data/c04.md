#### Exercise 02: ft_putnbr
* **Turn-in directory:** `ex02/`
* **Files to turn in:** `ft_putnbr.c`
* **Allowed functions:** `write`

Create a function that displays the number passed as a parameter. The function has to be able to display all possible values within an `int` type variable.
```c
void ft_putnbr(int nb);
```

> **Trap:** `INT_MIN` (-2147483648) cannot be negated. Handle it as a special case.

---

#### Exercise 03: ft_atoi
* **Turn-in directory:** `ex03/`
* **Files to turn in:** `ft_atoi.c`
* **Allowed functions:** None

Create a function that converts the initial portion of the string pointed to by `str` to `int` representation. The function must handle whitespace, `+` and `-` signs, just like the real `atoi`.
```c
int ft_atoi(char *str);
```

> **Trap:** Multiple signs: `"---+--+1234ab567"` → The real atoi skips whitespace, then counts signs: odd number of `-` means negative. Only `+` and `-` count, nothing else. If a non-sign non-digit appears after signs, return 0.

#### Exercise 04: ft_putnbr_base
* **Turn-in directory:** `ex04/`
* **Files to turn in:** `ft_putnbr_base.c`
* **Allowed functions:** `write`

Create a function that displays a number in a base system. The base is a string containing all unique characters that represent values from 0 upward.
```c
void ft_putnbr_base(int nbr, char *base);
```

> **Trap:** Validate the base: must be at least 2 chars, no duplicates, no `+` or `-`. If invalid, display nothing.

Create a function that converts a string argument to an `int`, using the given base.

int ft_atoi_base(char *str, char *base);

> **Trap:** Same base validation as `ft_putnbr_base`. Whitespace and sign handling like `ft_atoi`.
